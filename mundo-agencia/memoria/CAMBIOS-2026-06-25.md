## 6. ⚠️ Por qué el deploy de Netlify aún falla (acción tuya)

El log real de Netlify (deploy `6a3d40b9...`) dice:

> **"Build blocked: Unrecognized Git contributor. This plan allows only verified
> account members to push to private repos."** (`strict_contributor_verification_failure: true`)

El build se **bloquea en ~2 segundos, antes de compilar**. NO es el código (el build
local pasa limpio). Es una protección de tu cuenta de Netlify para repos privados: solo
deja desplegar commits de miembros verificados, y los commits de esta sesión llevan
co-autoría de Claude / los empuja la automatización, así que los marca como "contribuidor
no reconocido".

**Cómo arreglarlo (en tu panel de Netlify):**
- Proyecto `nexia-ia-com` → **Project configuration → Build & deploy → Deploy permissions**.
- Opción A: desactivar **"Strict contributor verification"**.
- Opción B: añadir tu identidad de Git (Michel8612 / tu email de commits) como **miembro
  verificado** del equipo de Netlify.

En cuanto lo desbloquees, el deploy compilará bien (lockfile y Next ya están arreglados).
