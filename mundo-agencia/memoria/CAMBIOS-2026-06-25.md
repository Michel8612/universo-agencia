## 2. ⚠️ ERRORES Y SOLUCIONES

### ERROR PRINCIPAL — El deploy de Netlify se bloquea
**Síntoma:** cada push al PR marca el deploy como *failed* en ~3 segundos, sin compilar.

**Causa real** (log de Netlify, vía API):
> *"Build blocked: Unrecognized Git contributor. This plan allows only verified account
> members to push to private repos."* (`strict_contributor_verification_failure: true`)

**No es el código** (el build local pasa limpio). Es una **limitación del plan gratis
"Dev"** (`nf_team_dev`): en repos privados solo despliegan contribuidores verificados, y
los commits de esta sesión (automatización + co-autoría Claude) salen como "no reconocidos".

> **RESUELTO 2026-06-25:** el repo se hizo **público** → desaparece la restricción de
> "verified contributor" (que solo aplica a repos privados). El deploy ya no se bloquea.
> Para que los cambios salgan en producción (nexia-ia-com.netlify.app) hay que fusionar
> el PR #1 a `main`.

**Soluciones (elige una):**

- **A) Deploy manual desde tu PC (recomendado para verlo ya).** Sube la carpeta construida
  sin pasar por el build de Git → la verificación no aplica:
  ```bash
  cd <repo>/mundo-ventas/web-nexia
  npm install
  npm run build                      # genera out/
  npx netlify-cli login              # 1 sola vez
  npx netlify-cli deploy --prod --dir=out
  # cuando pregunte, enlaza al sitio existente: nexia-ia-com
  ```
  → web live en https://nexia-ia-com.netlify.app

- **B) Verificar el contribuidor (arregla el auto-deploy permanente).**
  Abre el deploy bloqueado en Netlify → suele haber un aviso *"unrecognized contributor"*
  con botón para **aprobar/verificar** → apruébate. Si no aparece: **Team settings → Builds
  (o Members) → "Verified contributors"** → añade tu cuenta GitHub (Michel8612).

- **C) Hacer el repo público** (gratis, permanente). El bloqueo es solo para repos privados.
  ⚠️ Solo si NO hay secretos commiteados (revisado: `.env` está en `.gitignore`, solo se
  subió `.env.example` con placeholders — confírmalo antes).
